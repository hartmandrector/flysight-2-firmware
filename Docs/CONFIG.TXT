; Bluetooth BLE Divider Settings
;
; These dividers control the rate at which sensor data is transmitted
; over Bluetooth Low Energy. The BLE transmission rate equals:
;
;   BLE_Rate = Hardware_Rate / Divider
;
; Setting to 0 enables auto-calculation (recommended), which tries to run
; sensors at full ODR (divider=1). If total bandwidth would exceed the safe
; limit (1500 bytes/sec), auto sensors are scaled by percentage to fit.
;
; Priority order:
;   1. GPS rate (controlled by 'Rate' parameter, no divider)
;   2. Manual dividers (non-zero values, respected exactly)
;   3. Auto dividers (0 values, calculated to fit remaining budget)
;
; This approach respects your ODR choices - sensors run at full rate unless
; bandwidth requires throttling. Manual dividers take priority, allowing you
; to ensure critical sensors get their desired rate.
;
; Examples:
;   Default config (GPS 5Hz, all sensors low ODR) → All dividers = 1 (full rate)
;   High IMU (416 Hz accel/gyro) → Scales to fit, maintains relative priorities
;   Mixed manual/auto → Manual respected, auto scales to fit remaining budget
;
; Minimum rate: Auto-calculation enforces 0.5 Hz minimum on all sensors
;               to keep data flowing even under extreme bandwidth pressure.
;
; WARNING: Setting manual dividers too low may exceed BLE bandwidth limits,
;          causing buffer overrun and connection loss. Auto (0) is strongly
;          recommended for reliable operation.
;
BLE_Baro_Divider:  0 ; Auto-calculate (recommended)
BLE_Hum_Divider:   0 ; Auto-calculate (recommended)
BLE_Accel_Divider: 0 ; Auto-calculate (recommended)
BLE_Gyro_Divider:  0 ; Auto-calculate (recommended)
